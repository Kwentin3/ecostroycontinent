# Change Report

Документ: Content Inventory & Evidence Register v0.1  
Дата: 2026-03-23

## Где launch-core уже достаточно обеспечен материалами

- Канон запуска хорошо обеспечен рамкой: region recommendation, page map, launch-core pages и owner review logic уже зафиксированы в действующих docs.
- Минимальная trust base уже есть: юридическое лицо, city anchor `Сочи`, базовые регистрационные сведения и SRO membership.
- `About` и часть service pages можно начинать готовить как skeleton drafts после короткого owner session, потому что хотя бы общая business identity уже подтверждается.

## Где PRD / SEO intent опережают реальную фактуру

- Почти весь case layer: в доступном workspace нет готовых case packs.
- `Строительство гостевых домов` и особенно `Строительство коммерческих объектов`: intent есть, но доказательная база не показана.
- `Фасадные работы`: intent есть, но визуальная и proof base фактически отсутствует.
- Supporting articles легко скатятся в generic SEO text, если не собрать реальные service facts и examples.

## Какие 3-5 gaps сейчас самые опасные для запуска

- Нет подтверждённого photo / visual archive.
- Нет ни одного полноценно подтверждённого case pack.
- Нет подтверждённого публичного контактного набора для `/contacts`.
- Нет owner-approved claim boundaries по срокам, стоимости, гарантиям и scope.
- Нет publishable testimonials / client quotes.

## Что сознательно не включено в inventory

- Будущие non-phase-1 pages и surfaces.
- Полный asset taxonomy beyond launch need.
- Подробный DAM / CMS / implementation blueprint.
- Широкий blog backlog beyond minimum supporting contour.
- Неподтверждённые external review sources и speculative materials, которых нет в доступном inventory.
