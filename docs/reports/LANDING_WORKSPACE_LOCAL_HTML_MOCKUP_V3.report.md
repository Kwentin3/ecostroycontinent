# LANDING_WORKSPACE_LOCAL_HTML_MOCKUP_V3

## Scope
Собран новый локальный self-contained mockup `v3` для оценки Track 1 идеи. Главное изменение: в центре больше нет отдельного composition rail. Вместо этого оставлена одна центральная интерактивная поверхность страницы.

## Deliverables
- `artifacts/landing-workspace-mockup-v3.html`
- `docs/reports/LANDING_WORKSPACE_LOCAL_HTML_MOCKUP_V3.report.md`

## What Changed Compared To V2
Главное изменение относительно `v2`:
- отдельный composition rail полностью удалён;
- центр больше не разделён на “редактор слева + preview справа” внутри одной зоны;
- одна и та же структура страницы больше не показывается дважды;
- все действия по композиции происходят прямо на странице;
- connective copy стал ещё легче и ближе к inline-мостикам, а не к форме;
- правый rail дополнительно упрощён и больше не пытается играть роль شبه-инспектора.

## How The Rail Was Removed
В `v2` proof-блоки показывались и в rail, и в preview. Это создавало дублирование смысла.

В `v3` rail удалён полностью.

Теперь:
- материалы живут только слева;
- сама страница живёт только в центре;
- действия по блоку происходят прямо на выбранной секции страницы.

## Single Interactive Center Surface
Центр перестроен как одна страница-поверхность.

На ней можно:
- выбрать блок прямо в месте чтения;
- поменять bounded emphasis прямо на секции;
- переставить proof-блок выше/ниже прямо на секции;
- удалить блок прямо из секции;
- переключиться в более чистый `Предпросмотр`, где часть compose-controls скрывается.

Это делает центр единственной рабочей сценой и убирает проблему двойной интерпретации.

## Materials Usage Representation
Слева библиотека показывает только reusable материалы:
- `media`
- `services`
- `cases`

Для каждого материала есть простые состояния:
- `Свободно`
- `Добавлено`
- `главный`, если оператор явно сделал его главным proof.

Сложной системы “где именно используется” по-прежнему нет, и это сознательное упрощение.

## Connective Copy
Connective copy в `v3` сделан легче, чем в `v2`:
- как короткая `Связка` перед блоком;
- с маленькой inline-кнопкой `Изменить`;
- с компактным однострочным editor только для выбранного блока.

Это ближе к ощущению “я пишу glue текста страницы”, а не “я заполняю форму”.

## Right Column Simplification
Правая колонка стала ещё тише.

В ней оставлены только:
- компактный блок `Проверка`;
- `Передать на проверку`;
- маленький блок `AI-помощь`;
- минимальный контекст выбранного блока, только когда это реально полезно.

Она не конкурирует с центром и не выглядит как verification console.

## Helper Representation
Полного чата нет.

Helper представлен как:
- две маленькие bounded actions: `Подсказать связку` и `Сделать короче`;
- один компактный helper drawer с короткими подсказками.

Это подчёркивает, что helper вторичен по отношению к композиции страницы.

## Intentionally Left Out
Сознательно не включались:
- backend и сохранение;
- реальные generate/review маршруты;
- технические runtime-детали;
- свободный canvas;
- arbitrary styling;
- full chat;
- отдельный mirrored composition rail.
